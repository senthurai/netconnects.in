Savan compared to Maven

Maven is a legend

If your project have Maven configuration, it will load jars for the missing project reference by minor configuration changes.

If your's is not then you no need to migrate to maven. and still you can start creating many eclipse projects for developments

Efficient Savan


Less Wait Time
Again if number of project reference are more in number and you are going to setup the workspace on weekly basis production break fixes, it will be a painful activity.

Savan is to reduce the wait time for loading/updating those projects, refreshing, building, cleaning, deploying, staring the server. these effort are less for jar references when compared to project references in eclipse/IBM RAD projects.
